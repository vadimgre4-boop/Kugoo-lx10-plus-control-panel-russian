# Kugoo-lx10-plus-control-panel-russian
Kugoo LX10 Plus Control — веб-версия (Web Bluetooth)
Один файл index.html — та же функциональность, что и во Flutter-версии,
но без установки: открывается прямо в Chrome на Android.
Что нужно, чтобы запустить
Web Bluetooth работает только в "защищённом контексте" — HTTPS или
localhost. Просто открыть файл двойным тапом (file://) не получится,
Chrome заблокирует доступ к Bluetooth. Нужен реальный хостинг — для
статического HTML это буквально один файл, никакой сборки не требуется.
Да, GitHub + Vercel — стандартный и самый удобный вариант
Создай репозиторий на GitHub (публичный или приватный — не важно).
Залей туда index.html как есть, в корень репозитория (веб-интерфейс
GitHub → Add file → Upload files, либо git push).
Зайди на vercel.com → Add New → Project → Import Git Repository →
выбери репозиторий.
Vercel сам определит, что это статический HTML — никаких настроек
сборки не нужно, жми Deploy.
Получишь ссылку вида https://твой-репозиторий.vercel.app — она уже
на HTTPS, Bluetooth будет работать.
Дальше просто: правишь index.html в репозитории (прямо на GitHub
можно через встроенный редактор, или git push с компьютера) — Vercel
сам пересобирает и обновляет ссылку при каждом коммите. Это удобно
именно для этапа подбора протокола: поправил байты → закоммитил →
через ~10 секунд уже можно проверять на телефоне.
Открывай задеплоенную ссылку в Chrome на Android (Safari/iOS Web
Bluetooth не поддерживает в принципе, это ограничение Apple, не наше).
Чем веб-версия отличается от Flutter-версии
Родной выбор устройства. Не нужен свой экран сканирования — кнопка
"Подключить" сразу открывает системный список Bluetooth-устройств
браузера.
Важное ограничение Web Bluetooth: страница получает доступ только
к тем GATT-сервисам, чьи UUID заранее перечислены в коде
(Protocol.defaultOptionalServices()) — в отличие от нативного
приложения, которое видит вообще все сервисы устройства. Когда узнаешь
реальный UUID сервиса Kugoo — впиши его на вкладке "Отладка" в поле
"Доп. UUID сервисов" (там же есть пояснение) и переподключись.
Связь живёт, пока открыта вкладка/PWA и телефон не заблокирован —
свернул браузер или погас экран, соединение с высокой вероятностью
оборвётся. Для панели, которую держат перед глазами во время катания,
это не критично.
История поездок и сохранённые доп.UUID хранятся в localStorage
браузера (это нормально для обычного сайта, не для Claude-артефакта) —
очистка данных сайта в браузере их сотрёт.
Протокол — по-прежнему заглушка
Ровно так же, как в Flutter-версии: реальных байт команд LX10 Plus нигде
нет, Protocol в <script> — образец структуры (комментарий в начале
скрипта это явно помечает). Дальнейший путь такой же: снимаешь трафик
через nRF Connect + Bluetooth HCI snoop log при пользовании официальным
приложением, присылаешь дамп — разбираем и переписываем Protocol
внутри index.html.
Функции (без изменений от Flutter-версии)
Сканирование/подключение, панель (скорость/батарея/пробег/мощность),
Eco/Normal/Sport, круиз-контроль, фара, блокировка, лимит скорости,
полицейский режим (25 км/ч + попытка 250 Вт), плавность старта,
рекуперация, история поездок, вкладка "Отладка" (дерево GATT, ручная
отправка hex, живой лог). Автообновление прошивки намеренно не
реализовано — риск повредить контроллер без официального загрузчика.
